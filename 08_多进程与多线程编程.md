## 🎭 进程和线程——程序的"分身术"

### 🤔 从"多任务"说起

想象你正在用电脑：
- 🌐 浏览器在上网查资料
- 🎵 音乐软件在播放歌曲
- 📝 Word 在编辑文档

**这就是"多任务"** —— 操作系统同时运行多个程序！

#### 🧠 单核 CPU 如何实现"多任务"？

```
时间轴 →

CPU: [任务1 ▓▓] [任务2 ▓▓] [任务3 ▓▓] [任务1 ▓▓] [任务2 ▓▓] ...
      0.01秒   0.01秒   0.01秒   0.01秒   0.01秒

看起来: [任务1 ▓▓▓▓▓▓] 
        [任务2 ▓▓▓▓▓▓]  ← 好像同时在运行！
        [任务3 ▓▓▓▓▓▓]
```

> 💡 **秘密**：CPU 超快地在任务间"切换"，快到我们感觉不出来！

---

### 📚 核心概念：进程 vs 线程

#### 🏭 进程（Process）—— 独立的"工厂"

**定义**：一个运行中的程序就是一个进程

| 例子 | 说明 |
|------|------|
| 打开浏览器 | 启动一个浏览器进程 |
| 打开记事本 | 启动一个记事本进程 |
| 打开两个记事本 | 启动**两个**独立的记事本进程 |

**特点**：
- 🏠 **独立空间**：每个进程有自己的内存空间
- 🔒 **互不干扰**：一个进程崩溃不会影响其他进程
- 💰 **资源开销大**：创建和切换成本高

---

#### 🧵 线程（Thread）—— 工厂里的"工人"

**定义**：进程内的"子任务"，共享进程的内存空间

| 例子 | 说明 |
|------|------|
| Word 打字 | 一个线程 |
| Word 拼写检查 | 另一个线程 |
| Word 自动保存 | 又一个线程 |

**特点**：
- 🏢 **共享空间**：同一进程内的线程共享内存
- ⚡ **轻量级**：创建和切换成本低
- ⚠️ **风险共存**：一个线程崩溃可能导致整个进程崩溃

---

### 🎯 进程 vs 线程 对比表

| 特性 | 进程（Process） | 线程（Thread） |
|------|-----------------|----------------|
| **比喻** | 独立的工厂 | 工厂里的工人 |
| **内存** | 独立内存空间 | 共享进程内存 |
| **通信** | 需要 IPC（管道、队列等） | 直接读写共享变量 |
| **开销** | 大（创建慢、切换慢） | 小（创建快、切换快） |
| **安全** | 高（互不影响） | 低（共享内存需同步） |
| **崩溃影响** | 只影响自己 | 可能导致整个进程崩溃 |

## 🏭 多进程编程——multiprocessing 模块

### 🤔 什么时候用多进程？

想象你有 16 个任务，每个任务需要 2 秒：

| 方式 | 执行时间 | 说明 |
|------|----------|------|
| 单进程串行 | 32 秒 | 一个一个执行 |
| 16 进程并行 | ~2 秒 | 同时执行（需16核） |
| 8 进程并行 | ~4 秒 | 8个一组，分两组执行 |

> 💡 **适用场景**：CPU 密集型任务（计算量大，需要大量 CPU 运算）

---

### 🚀 创建进程的基本步骤

```python
import multiprocessing
import time

# 1️⃣ 定义任务函数
def task():
    print(f'{time.strftime("%H:%M:%S")} 开始执行任务...')
    time.sleep(2)  # 模拟耗时操作
    print(f'{time.strftime("%H:%M:%S")} 任务完成！')

# 2️⃣ 创建并启动进程
if __name__ == "__main__":  # ⚠️ Windows 必须加这行！
    for i in range(4):
        # 创建进程
        p = multiprocessing.Process(target=task)
        # 启动进程
        p.start()
    
    print("主进程继续执行...")
```

**⚠️ 重要提示**：`if __name__ == "__main__":` 在 Windows 上**必须加**！
- 原因：Windows 创建进程会复制主进程代码，不加会造成**无限递归创建子进程**

---

### 📋 Process 类详解

#### 构造函数参数

```python
Process(
    group=None,        # 进程组，一般不用
    target=None,       # 目标函数（要执行的任务）
    name=None,         # 进程名，默认 Process-1, Process-2...
    args=(),           # 传递给 target 的位置参数（元组）
    kwargs={}          # 传递给 target 的关键字参数（字典）
)
```

#### 常用方法和属性

| 方法/属性 | 说明 | 示例 |
|-----------|------|------|
| `start()` | 启动进程 | `p.start()` |
| `join()` | 等待进程结束 | `p.join()` 或 `p.join(5)` 最多等5秒 |
| `is_alive()` | 进程是否还在运行 | `p.is_alive()` |
| `terminate()` | 强制终止进程 | `p.terminate()` |
| `name` | 进程名称 | `p.name` |
| `pid` | 进程 ID | `p.pid` |

---

### 🎯 带参数的进程

```python
import multiprocessing
import random
import time

def task(task_id, duration):
    """带参数的任务函数"""
    print(f'📝 任务{task_id}开始，预计耗时{duration}秒')
    time.sleep(duration)
    print(f'✅ 任务{task_id}完成！')

if __name__ == "__main__":
    for i in range(5):
        # args 必须是元组，单个元素要加逗号！
        duration = random.randint(1, 3)
        p = multiprocessing.Process(
            target=task, 
            args=(i, duration)  # 注意：(i,) 不是 (i)
        )
        p.start()
```

> 💡 **注意**：`args=(i,)` 是元组，逗号不能少！`args=(i)` 会被当成整数。

## 🏊 进程池 Pool——更高效的进程管理方式

### 🤔 为什么要用进程池？

想象一个水池场景：

```
┌─────────────────────────────────────────────────────────┐
│                      进程池（Pool）                      │
│  ┌─────────────────────────────────────────────────┐   │
│  │              水池（最多3个水龙头）                │   │
│  │                                                 │   │
│  │   💧        💧        💧                        │   │
│  │  [任务1]   [任务2]   [任务3]   ← 同时执行3个      │   │
│  │                                                 │   │
│  │  任务4 □  任务5 □  任务6 □ ...  ← 排队等待        │   │
│  │                                                 │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  优势：                                                  │
│  ✅ 复用进程，不用频繁创建销毁                            │
│  ✅ 控制并发数量，避免系统过载                            │
│  ✅ 自动管理任务队列                                      │
└─────────────────────────────────────────────────────────┘
```

> 💡 **比喻**：进程池就像一个有固定数量水龙头的公共水池，大家排队接水，接完一个下一个上！

![Image 1](./images/chapter08_01.jpeg)

---

### 📋 Pool 类常用方法

| 方法 | 说明 | 特点 |
|------|------|------|
| `apply_async(func, args)` | **非阻塞**方式调用 | ⭐ 推荐！并行执行 |
| `apply(func, args)` | **阻塞**方式调用 | 等待上一个完成才执行下一个 |
| `close()` | 关闭进程池 | 不再接受新任务 |
| `terminate()` | 立即终止 | 强制结束所有进程 |
| `join()` | 等待所有进程完成 | 必须在 `close()` 后使用 |

> 💡 **阻塞 vs 非阻塞**：
> - **阻塞**：必须等任务1完成，任务2才能开始
> - **非阻塞**：任务1、2、3 同时开始，谁快谁先结束

---

### 🎯 进程池实战代码

```python
from multiprocessing import Pool
import os
import time

def task(task_id):
    """任务函数"""
    print(f"📝 任务{task_id} → 进程ID: {os.getpid()} 开始执行")
    time.sleep(2)  # 模拟耗时操作
    print(f"✅ 任务{task_id} → 完成！")
    return f"任务{task_id}的结果"

if __name__ == "__main__":
    print("🚀 创建进程池（最大3个进程）")
    
    # 1️⃣ 创建进程池，最多同时运行3个进程
    pool = Pool(3)
    
    # 2️⃣ 提交10个任务
    print("📤 提交10个任务...")
    for i in range(10):
        # apply_async: 非阻塞方式（推荐）
        pool.apply_async(task, args=(i,))
    
    # 3️⃣ 关闭进程池（不再接受新任务）
    pool.close()
    
    print("⏳ 等待所有任务完成...")
    
    # 4️⃣ 等待所有进程执行完毕
    pool.join()
    
    print("🎉 所有任务已完成！")
```

**运行结果**：
```
🚀 创建进程池（最大3个进程）
📤 提交10个任务...
⏳ 等待所有任务完成...
📝 任务0 → 进程ID: 12345 开始执行
📝 任务1 → 进程ID: 12346 开始执行
📝 任务2 → 进程ID: 12347 开始执行
✅ 任务0 → 完成！
📝 任务3 → 进程ID: 12345 开始执行  ← 复用进程
...
🎉 所有任务已完成！
```

---

### 🔍 代码解读

```python
# 关键步骤解析
pool = Pool(3)           # 创建最多3个进程的进程池
pool.apply_async(...)    # 非阻塞提交任务（并行执行）
pool.close()             # 关闭池子，不再接受新任务
pool.join()              # 等待所有任务完成（阻塞等待）
```

**注意执行顺序**：
- 任务 0、1、2 **同时开始**（因为有3个进程）
- 任务 3 必须等 0、1、2 中**任意一个完成**才能开始
- 以此类推，直到所有任务完成

---

### ⚡ 调整进程池大小

```python
# 默认使用 CPU 核心数
pool = Pool()  # 自动检测 CPU 核数

# 指定进程数
pool = Pool(4)   # 同时运行4个进程
pool = Pool(8)   # 同时运行8个进程
pool = Pool(16)  # 同时运行16个进程
```

> 💡 **建议**：进程数一般设置为 CPU 核心数或核心数的 1-2 倍

## 🧵 多线程编程——threading 模块

### 🤔 什么时候用多线程？

| 场景 | 推荐方案 | 原因 |
|------|----------|------|
| 下载多个文件 | 多线程 | IO 等待时 CPU 空闲 |
| 网络请求 | 多线程 | 等待服务器响应 |
| 文件读写 | 多线程 | IO 操作耗时 |
| 复杂计算 | 多进程 | 绕过 GIL，真正并行 |

> 💡 **适用场景**：IO 密集型任务（网络、文件操作等等待时间长的任务）

---

### 🚀 创建线程的基本步骤

```python
import threading
import time

# 1️⃣ 定义任务函数
def download(file_name):
    print(f'⬇️ 开始下载: {file_name}')
    time.sleep(2)  # 模拟下载耗时
    print(f'✅ 下载完成: {file_name}')

# 2️⃣ 创建并启动线程
if __name__ == '__main__':
    print(f"主线程: {threading.current_thread().name}")
    
    # 创建3个线程同时下载
    files = ['视频.mp4', '音乐.mp3', '文档.pdf']
    threads = []
    
    for file in files:
        # 创建线程
        t = threading.Thread(target=download, args=(file,))
        threads.append(t)
        # 启动线程
        t.start()
    
    # 等待所有线程完成
    for t in threads:
        t.join()
    
    print("🎉 所有下载完成！")
```

**运行结果**：
```
主线程: MainThread
⬇️ 开始下载: 视频.mp4
⬇️ 开始下载: 音乐.mp3
⬇️ 开始下载: 文档.pdf
✅ 下载完成: 视频.mp4
✅ 下载完成: 音乐.mp3
✅ 下载完成: 文档.pdf
🎉 所有下载完成！
```

### ⚠️ 重要：Python 的 GIL

```
┌─────────────────────────────────────────────────────────┐
│                    Python GIL 机制                       │
├─────────────────────────────────────────────────────────┤
│  GIL = Global Interpreter Lock（全局解释器锁）           │

```

### 🎯 线程安全问题

**问题**：多个线程同时修改同一个变量，会导致数据混乱！

```python
import threading
import time

# 共享变量
counter = 0

def add_one():
    global counter
    for _ in range(100000):
        # 这里有问题！多个线程同时读取和写入
        temp = counter
        time.sleep(0.000001)  # 模拟耗时
        counter = temp + 1

# 创建两个线程
thread1 = threading.Thread(target=add_one)
thread2 = threading.Thread(target=add_one)

thread1.start()
thread2.start()
thread1.join()
thread2.join()

print(f"预期结果: 200000")
print(f"实际结果: {counter}")  # 可能远小于 200000！
```

**解决方案**：使用锁（Lock）

```python
import threading

# 创建锁
lock = threading.Lock()
counter = 0

def add_one_safe():
    global counter
    for _ in range(100000):
        # 获取锁
        lock.acquire()
        try:
            counter += 1
        finally:
            # 释放锁（一定要释放！）
            lock.release()
        
        # 或者用 with 语句（推荐）
        # with lock:
        #     counter += 1
```

> 💡 **使用 with 语句更简洁，且自动释放锁，避免死锁**

## 🆚 多进程 vs 多线程——终极对比

### 🎯 Master-Worker 模式

多任务编程通常采用 **Master-Worker** 架构：
- **Master**（主进程/主线程）：负责分配任务
- **Worker**（子进程/子线程）：负责执行任务

```
┌─────────────────────────────────────────────────────────┐
│                    Master-Worker 模式                    │
├─────────────────────────┬───────────────────────────────┤
│      多进程模式          │         多线程模式             │
├─────────────────────────┼───────────────────────────────┤
│  ┌─────────┐            │  ┌─────────┐                  │
│  │ Master  │            │  │ Master  │                  │
│  │ Process │            │  │ Thread  │                  │
│  └────┬────┘            │  └────┬────┘                  │
│       │                 │       │                       │
│   ┌───┴───┐             │   ┌───┴───┐                   │
│   ▼   ▼   ▼             │   ▼   ▼   ▼                   │
│  🏭  🏭  🏭             │  🧵  🧵  🧵                   │
│ Worker  Worker  Worker  │ Worker  Worker  Worker        │
│ Process Process Process │ Thread  Thread  Thread        │
└─────────────────────────┴───────────────────────────────┘
```

---

### 📊 优缺点对比表

| 对比项 | 多进程 | 多线程 |
|--------|--------|--------|
| **稳定性** | ⭐⭐⭐⭐⭐ 高，进程崩溃互不影响 | ⭐⭐⭐ 低，一个线程崩溃整个进程挂 |
| **创建开销** | ⭐⭐ 大，创建慢 | ⭐⭐⭐⭐⭐ 小，创建快 |
| **内存占用** | ⭐⭐ 大，独立内存空间 | ⭐⭐⭐⭐⭐ 小，共享内存 |
| **数据共享** | ⭐⭐ 复杂，需要 IPC | ⭐⭐⭐⭐⭐ 简单，直接读写变量 |
| **CPU 利用率** | ⭐⭐⭐⭐⭐ 真正并行（多核） | ⭐⭐⭐ 受 GIL 限制 |
| **调试难度** | ⭐⭐⭐ 中等 | ⭐⭐ 较难，有竞态条件 |
| **适用场景** | CPU 密集型 | IO 密集型 |

### 🌰 实际案例

| 应用场景 | 推荐方案 | 原因 |
|----------|----------|------|

| 网络爬虫 | 多线程 | IO 等待多 |
| 视频转码 | 多进程 | 计算量大 |
| 文件下载器 | 多线程 | 网络 IO 多 |

---

### ⚠️ 重要提醒

```python
# ❌ 不要滥用多进程/多线程！

# 错误示范：创建10000个进程
for i in range(10000):
    p = multiprocessing.Process(target=task)
    p.start()  # 系统会崩溃！

# ✅ 正确做法：使用进程池控制数量
with multiprocessing.Pool(4) as pool:  # 最多4个进程
    pool.map(task, range(10000))
```

> 💡 **黄金法则**：
> 1. **能单线程就不多线程** —— 简单即是美
> 2. **需要并发首选线程池/进程池** —— 控制资源使用
> 3. **多线程注意加锁** —— 避免数据竞争
> 4. **先测试再优化** —— 不要过早优化

## 📝 文档总结

### 一、核心知识点回顾

#### 1. 进程（Process）——独立的"工厂"

**核心概念**：
- 🏭 进程是操作系统分配资源的基本单位
- 🏠 每个进程有独立的内存空间
- 🔒 进程之间互不影响，一个崩溃不影响其他进程

**创建方式**：
```python
from multiprocessing import Process

p = Process(target=函数名, args=(参数,))
p.start()  # 启动进程
p.join()   # 等待进程结束
```

**适用场景**：CPU 密集型任务（大量计算）

---

#### 2. 进程池（Pool）——更高效的进程管理

**核心概念**：
- 🏊 预先创建固定数量的进程，复用进程资源
- 🎯 控制并发数量，避免系统过载
- 📋 自动管理任务队列

**使用方法**：
```python
from multiprocessing import Pool

pool = Pool(3)              # 创建最多3个进程的池子
pool.apply_async(函数, args) # 非阻塞提交任务
pool.close()                # 关闭池子
pool.join()                 # 等待所有任务完成
```

---

#### 3. 线程（Thread）——工厂里的"工人"

**核心概念**：
- 🧵 线程是 CPU 调度的基本单位
- 🏢 同一进程内的线程共享内存空间
- ⚡ 创建和切换开销小

**创建方式**：
```python
from threading import Thread

t = Thread(target=函数名, args=(参数,))
t.start()  # 启动线程
t.join()   # 等待线程结束
```

**适用场景**：IO 密集型任务（网络、文件操作）

---

#### 4. GIL（全局解释器锁）

**关键概念**：
- 🔒 Python 的 GIL 保证同一时刻只有一个线程执行 Python 字节码
- 📊 多线程不能真正并行执行 CPU 密集型任务
- 💡 CPU 密集型任务请使用多进程绕过 GIL

---

#### 5. 线程安全与锁

**问题**：多个线程同时修改共享变量会导致数据混乱

**解决方案**：
```python
from threading import Lock

lock = Lock()

# 方式1：手动获取和释放
lock.acquire()
# 修改共享变量
lock.release()

# 方式2：使用 with 语句（推荐）
with lock:
    # 修改共享变量
    pass
```

---

### 二、多进程 vs 多线程 速查表

| 特性 | 多进程 | 多线程 |
|------|--------|--------|
| **内存空间** | 独立 | 共享 |
| **创建开销** | 大 | 小 |
| **数据共享** | 复杂（需 IPC） | 简单（直接读写） |
| **稳定性** | 高 | 低（一崩全崩） |
| **CPU 利用率** | 真正并行 | 受 GIL 限制 |
| **适用场景** | CPU 密集型 | IO 密集型 |
| **Python 模块** | `multiprocessing` | `threading` |

```
任务是 CPU 密集型？
    ├── 是 → 使用多进程
    └── 否 → 任务是 IO 密集型？
            ├── 是 → 使用多线程
            └── 否 → 单线程就够了
```

Thread(target=task, args=99)

import threading
import time

    # 持续3秒，每秒输出一次

        current_time = time.strftime('%H:%M:%S')

        time.sleep(1)


def movie(name):
    """播放视频"""
    # 持续10秒，每2秒输出一次（循环5次）
    for i in range(5):
        current_time = time.strftime('%H:%M:%S')
        print(f"{current_time}: 正在播放视频[{name}]")
        time.sleep(2)


if __name__ == '__main__':

**3. 实现一个多线程的爬虫程序，同时爬取多个网页的标题，要求如下：**
- 爬取 5 个网页的标题
- 使用多线程，每个网页一个线程
- 使用锁确保输出不会混乱
- 统计总耗时

> 💡 **提示**：使用 `requests` 库获取网页，使用 `threading.Lock()` 保护输出。

<details>
<summary>🔑 参考答案与解析（点击展开）</summary>

**解题思路**：
1. 导入必要的模块：`threading`、`requests`、`time`、`re`
2. 创建锁对象，保护输出
3. 定义爬取函数，接收 URL，获取网页并提取标题
4. 创建多个线程，每个线程爬取一个网页
5. 等待所有线程完成，统计总耗时

**代码实现**：

```python
import threading
import time
import re

# 尝试导入 requests，如果没有安装则给出提示
try:
    import requests
except ImportError:
    print("请先安装 requests 库: pip install requests")
    exit(1)

# 创建锁，保护输出
print_lock = threading.Lock()


def crawl_title(url, thread_id):
    """
    爬取网页标题
    
    参数:
        url: 网页地址
        thread_id: 线程编号
    """
    thread_name = threading.current_thread().name
    
    with print_lock:
        print(f"🕷️ [线程{thread_id}-{thread_name}] 开始爬取: {url}")
    
    try:
        # 发送请求
        start_time = time.time()
        response = requests.get(url, timeout=10)
        response.raise_for_status()
        
        # 提取标题（简单的正则匹配）
        html = response.text
        title_match = re.search(r'<title>(.*?)</title>', html, re.IGNORECASE)
        title = title_match.group(1) if title_match else "未找到标题"
        
        # 清理标题中的空白字符
        title = title.strip().replace('\n', ' ').replace('\r', '')
        
        elapsed_time = time.time() - start_time
        
        with print_lock:
            print(f"✅ [线程{thread_id}] 完成！耗时{elapsed_time:.2f}秒")
            print(f"   标题: {title[:50]}{'...' if len(title) > 50 else ''}")
            print()
        
        return title
        
    except requests.RequestException as e:
        with print_lock:
            print(f"❌ [线程{thread_id}] 爬取失败: {e}\n")
        return None


def main():
    """主函数"""
    # 要爬取的网页列表
    urls = [
        "https://www.example.com",
        "https://www.python.org",
        "https://www.github.com",
        "https://stackoverflow.com",
        "https://www.wikipedia.org",
    ]
    
    print("=" * 60)
    print("🕷️ 多线程网页爬虫")
    print("=" * 60)
    print(f"📄 目标网页数: {len(urls)}")
    print(f"🔧 线程数: {len(urls)}（每个网页一个线程）")
    print("=" * 60)
    print()
    
    # 记录开始时间
    start_time = time.time()
    
    # 创建线程列表
    threads = []
    
    # 创建并启动线程
    for i, url in enumerate(urls, 1):
        t = threading.Thread(
            target=crawl_title, 
            args=(url, i),
            name=f"Crawler-{i}"
        )
        threads.append(t)
        t.start()
    
    # 等待所有线程完成
    for t in threads:
        t.join()
    
    # 计算总耗时
    total_time = time.time() - start_time
    
    print("=" * 60)
    print(f"🎉 爬取完成！")
    print(f"⏱️ 总耗时: {total_time:.2f} 秒")
    print(f"📊 平均每个网页: {total_time/len(urls):.2f} 秒")
    print("=" * 60)


if __name__ == '__main__':
    main()
```
